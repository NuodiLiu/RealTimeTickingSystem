# Device Name Editing Feature

## 功能概述
在设备列表中，每个设备卡片右上角的下拉菜单现在包含"Edit Name"选项，允许用户直接编辑设备名称。

## 🎯 关键交互特性：

1. **一键进入编辑模式**: 点击"Edit Name" → 立即进入编辑状态
2. **自动焦点**: 文本框自动获得焦点并选中文本
3. **点击外部确认**: 符合用户直觉的交互模式
4. **键盘快捷键**: Enter保存、Escape取消
5. **简洁设计**: 无额外按钮，保持界面整洁
6. **实时同步**: 通过WebSocket实时更新所有客户端

## 视觉反馈

### 编辑状态指示器
- 编辑模式下，名称显示为带底边框的文本框
- 文本框获得蓝色焦点边框
- 简洁的设计，无额外的按钮干扰

### Toast通知
- 保存时显示加载状态："Updating device name to "新名称"..."
- 成功时显示："Device name has been successfully updated to "新名称"."
- 失败时显示错误信息

## 技术实现

### 前端
1. **DeviceCard组件**: 包含编辑状态管理和UI交互
2. **DeviceList组件**: 传递编辑回调函数
3. **DevicesSection组件**: 处理API调用和错误处理

### 后端
1. **API端点**: `PATCH /api/devices/:id/name`
2. **WebSocket通知**: 实时更新所有连接的dashboard客户端
3. **权限验证**: 需要员工认证token

### 实时更新
- 设备名称更改后，通过WebSocket自动更新所有连接的客户端
- 无需手动刷新页面即可看到更改结果

## 约束条件

### 输入验证
- 设备名称不能为空
- 前后空格会被自动trim
- 如果新名称与当前名称相同，不会发起API请求

### 权限要求
- 需要有效的员工认证
- 只有已认证的员工可以编辑设备名称

### 错误处理
- 网络错误时显示相应的错误toast
- API错误时显示具体错误信息
- 编辑状态会在错误时自动退出

## 使用场景

1. **设备初始化**: 为新配对的设备设置有意义的名称
2. **设备重新部署**: 当设备移动到新位置时更新名称
3. **组织管理**: 按照统一的命名规范管理设备
4. **故障排查**: 通过清晰的设备名称快速定位问题设备

## 示例命名规范

建议的设备命名格式：
- `前台-反馈设备-01`
- `注册处-iPad-A`
- `咨询台-反馈终端`
- `楼层2-注册设备`

这样的命名可以帮助工作人员快速识别设备的位置和用途。
